# Environment Variables Setup Guide

Create a file called `.env` inside the `server/` folder.  
**Never commit this file to Git — it is in `.gitignore`.**

---

## server/.env

```env
# ─── Database ──────────────────────────────────────────────────────────────────
# Get this from: Supabase Dashboard → Project Settings → Database → Connection string (URI)
# Use the "Transaction" mode URI (port 6543), not direct (5432) unless on a VPS.
DATABASE_URL=postgresql://postgres.[project-ref]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres

# ─── JWT Secrets ───────────────────────────────────────────────────────────────
# Generate with: node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
JWT_SECRET=your_64_char_hex_string_here
JWT_REFRESH_SECRET=another_64_char_hex_string_here

# ─── Supabase Storage ──────────────────────────────────────────────────────────
# Get these from: Supabase Dashboard → Project Settings → API

# Project URL — looks like: https://abcdefghijklmn.supabase.co
SUPABASE_URL=https://your-project-ref.supabase.co

# Service Role Key (NOT the anon key) — found under Project Settings → API → service_role
# This key bypasses Row Level Security — keep it SECRET, server-side only.
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...your_service_role_key

# ─── Email (Resend) ────────────────────────────────────────────────────────────
# Get this from: https://resend.com/api-keys
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxx

# From address — must be a verified domain in Resend
RESEND_FROM_EMAIL=noreply@yourdomain.com

# ─── App ───────────────────────────────────────────────────────────────────────
NODE_ENV=development
PORT=3001

# Public URL of your frontend (used for application links)
FRONTEND_URL=http://localhost:3000
```

## client/.env.local

Create this in the `client/` folder:

```env
# Backend API URL
NEXT_PUBLIC_API_URL=http://localhost:3001/api

# Public frontend URL (used for application links)
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

## How to find your Supabase values

### SUPABASE_URL
1. Go to **https://supabase.com/dashboard**
2. Open your project
3. Click **Project Settings** (gear icon) → **API**
4. Copy the **Project URL** — it looks like `https://abcdefg.supabase.co`

### SUPABASE_SERVICE_ROLE_KEY
1. Same page: **Project Settings → API**
2. Scroll to **Project API keys**
3. Copy the **service_role** key (click "Reveal")
4. ⚠️ This is a secret — it bypasses all Row Level Security. Never expose it client-side.

### DATABASE_URL
1. Go to **Project Settings → Database**
2. Click **Connection string** → choose **URI**
3. Switch the mode to **Transaction** (port 6543) for serverless/pooled connections
4. Copy and replace `[YOUR-PASSWORD]` with your database password

## Creating Supabase Storage Buckets

You need to create these 5 buckets in your Supabase project:

1. Go to **Storage** in your Supabase dashboard
2. Click **New bucket** for each:

| Bucket Name | Public? | Use |
|---|---|---|
| `documents` | ❌ Private | Tenant ID uploads, misc |
| `leases` | ❌ Private | Lease PDF files |
| `receipts` | ❌ Private | Payment receipt PDFs |
| `statements` | ❌ Private | Owner statement PDFs |
| `branding` | ✅ Public | Logo and branding images |

All buckets except `branding` should be **private** — files are served via signed URLs (1-hour expiry) generated by the server using the service role key.

### RLS Policy for private buckets
For each private bucket, add this RLS policy so only your server (using service role key) can access files:

```sql
-- In Supabase SQL Editor, run for each bucket:
-- The service_role key bypasses RLS entirely, so private buckets
-- just need to deny anon/authenticated direct access.
-- No additional policy needed — service_role bypasses RLS by design.
```

The `SUPABASE_SERVICE_ROLE_KEY` bypasses RLS automatically on the server, so no extra policies are needed for server-side operations.

## Redis (optional — for rate limiting / cron)

If you use Redis for job queues or rate limiting:

```env
REDIS_URL=redis://localhost:6379
# or for Upstash:
REDIS_URL=rediss://default:password@us1-xxx.upstash.io:6380
```

## Quick start

```bash
# 1. Copy this template
cp server/.env.example server/.env   # if .env.example exists

# 2. Fill in your values (see above)
nano server/.env

# 3. Run database migrations
cd server && npx prisma migrate dev

# 4. Start the server
npm run dev

# 5. In another terminal, start the client
cd ../client && npm run dev
```
